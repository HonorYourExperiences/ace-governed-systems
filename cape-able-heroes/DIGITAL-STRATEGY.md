# Cape-Able Heroes – Digital Strategy (v1)

**Purpose:** Govern the evaluation, decision-making, and development of digital components for Cape-Able Heroes — ensuring that any digital investment is grounded in field evidence and axiomatic alignment, not feature excitement or trend-chasing.  
**Status:** v1 — evaluation framework active. No digital components built yet.

---

## Governing Principle

**Do not build digital tools because they seem like a good idea.**

Build them only when field evidence answers yes to all three of the following questions:

1. **Does this digital component extend the core transformation** (hesitation → participation) in a way that cannot be done as well — or at all — with physical materials?
2. **Does it serve the child or parent directly**, rather than primarily reducing facilitator workload or improving optics?
3. **Is there demonstrated demand** — meaning real customers have asked for it, or field data shows a clear gap that a digital tool would close?

If all three are yes, evaluate it. If any are no, do not build it yet — log the idea in the Decision Log below and return to it when evidence accumulates.

---

## Why This Framework Exists

Digital tools carry a specific risk for Cape-Able Heroes: they can create the appearance of progress (SAGA Theater) while distancing the program from what actually works — a human, present, witnessing a child take a real step.

A digital tool that feels impressive but reduces the quality of the identity moment or the witnessing model is not an asset. It is a governance failure.

This document exists to prevent that.

---

## Digital Component Categories

The following categories of digital tools have been identified as potentially relevant. Each is evaluated against the three-question framework when evidence warrants.

### Category A — Session Support Tools

Tools that facilitators or participants use during a live WonderLab session.

**Examples:** digital Hero Card generator, printable materials builder, theme library, observation tool app

**Framework check:**
- Extends core transformation? Uncertain — the physical artifact (Hero Card, badge) has demonstrated value
- Serves child/parent directly? Potentially for personalization
- Demonstrated demand? Not yet

**Status:** Not yet evaluated. Revisit after 5+ paid activations.

---

### Category B — At-Home Continuation

Tools that parents and children use between or after sessions to continue the identity-building work.

**Examples:** digital mission cards, weekly "cape-able challenge" prompts, parent reflection app, audio/video story components

**Framework check:**
- Extends core transformation? Yes — parents consistently ask "what do I do after the session?"
- Serves child/parent directly? Yes — this is directly for the family
- Demonstrated demand? Partial — field evidence from Phase 1 suggests parents want continuation tools

**Status:** Moderate priority. Evaluate after Phase 3 with specific parent feedback data.

---

### Category C — Educator & Facilitator Tools

Tools that reduce prep time, improve delivery consistency, or support certified facilitators running sessions independently.

**Examples:** session prep app, digital observation logging, facilitator community platform

**Framework check:**
- Extends core transformation? No directly — facilitator efficiency tools do not reach the child
- Serves child/parent directly? No — serves the facilitator
- Demonstrated demand? Not yet — only relevant once 5+ external facilitators are certified

**Status:** Low priority in Phase 4. Revisit when licensing program has 5+ active Tier 1 facilitators.

---

### Category D — Digital Products

Standalone digital products sold directly to consumers — not tied to a live session.

**Examples:** digital storybook, audio experience, e-learning module for parents, app for children

**Framework check:**
- Extends core transformation? Possibly — depends on the specific product design
- Serves child/parent directly? Yes if well-designed
- Demonstrated demand? Unknown — no testing has occurred

**Status:** Do not build in Phase 4 without field evidence of demand. Log interest signals in the Decision Log below.

## Build Decision Framework

When an idea reaches the "Evaluate now" stage, use this framework before committing resources.

### Step 1 — Define the Minimum Version
What is the smallest possible version of this tool that would produce evidence about whether it works? Do not build the full vision first. Build the minimum.

### Step 2 — Define the Evidence It Will Produce
What specific question does this digital component answer? What would "it worked" look like? Write this before building anything.

### Step 3 — Estimate the Cost
Time, money, and complexity. If the minimum version costs more than 2 weeks of focused work or more than $500, consider whether a lower-fidelity test can produce the same evidence.

### Step 4 — SAGA Proposal Required
Any digital component that moves to active development must be filed as a SAGA proposal with the three-question answers documented. No exceptions.
